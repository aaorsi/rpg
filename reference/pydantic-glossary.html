Pydantic Glossary

RPG policy orchestrator — quick reference

Base classes

CamelModel
Base with alias_generator=to_camel, populate_by_name=True, extra="ignore". Used for LLM outputs and API responses — unknown JSON keys are dropped silently.
StrictCamelModel
Same camelCase wire format, but extra="forbid". Used for inbound requests from Unity — unknown fields raise ValidationError.

Key models

DialogueTurnRequest
POST /v1/dialogue/turn body. Bundles NpcContext + TurnContext.
LlmDialogueOutput
Single source of truth for the shape the LLM must emit. Drives dialogue_turn_result.schema.json.
SessionNarrativeCanon
Canonical session narrative. Drives session_narrative.schema.json.
PolicyEnvelope
Uniform success/failure wrapper. @model_validator enforces: ok=true requires a payload; ok=false requires error.

Operations

model_validate(obj)
Parse a dict (or compatible object) into a typed model. Raises ValidationError on bad data. Used after parse_dialogue_output() coerces messy LLM text.
model_dump(by_alias=True)
Serialize to dict with camelCase keys for JSON wire format.
model_json_schema(by_alias=True)
Emit JSON Schema for Unity and documentation. Called by schema_export.build_schemas().

Pipeline terms

SoT (single source of truth)
app/models.py — committed *.schema.json files are generated artifacts, not hand-edited.
Schema sync test
tests/test_schema_sync.py — fails CI if generated schemas drift from models. Fix: python scripts/generate_schemas.py.
Tolerant parse
dialogue_parsing.py — strips markdown fences, normalizes key synonyms, then hands a clean dict to LlmDialogueOutput.model_validate.
